# Personal Portfolio (Jekyll + GitHub Pages)

Academic portfolio for Seungeon Lee, hosted at [flansma.github.io](https://flansma.github.io).

Built with Jekyll on GitHub Pages with rich SEO and structured data:
- Per-content **JSON-LD** (`Person`, `WebSite`, `Event`, `ScholarlyArticle`, `SoftwareSourceCode` + `ResearchProject`, `BreadcrumbList`)
- **Google Scholar** indexing via Highwire `citation_*` meta tags
- **ORCID** identifier wired into the Person graph (sameAs + identifier PropertyValue)
- Open Graph article tags, Twitter cards, canonical, sitemap, Atom feed, PWA manifest
- Multi-language UI (EN / JA / KO) via JS i18n (full static i18n is a planned next step)

## Project structure

```
├── _config.yml                # Jekyll config + plugins + collection layout defaults
├── _data/
│   ├── profile.yml            # Identity: name, ORCID, affiliation, alma_mater, social
│   ├── bio.yml                # Positions / timeline
│   ├── navigation.yml         # Header nav
│   ├── scholarships.yml, skills.yml, experience.yml
├── _includes/
│   ├── head.html              # head with {% seo %}, citation_meta, article_meta, jsonld
│   ├── header.html, footer.html
│   ├── biblio-item.html       # Bibliography list item (auto-routes to local detail page)
│   ├── citation_meta.html     # Highwire `citation_*` meta for academic pages
│   ├── article_meta.html      # OpenGraph article:* meta for content pages
│   ├── social-icon.html       # SVG icons (linkedin/x/github/orcid/scholar/researchgate)
│   └── jsonld/
│       ├── person.html        # Homepage Person + WebSite
│       ├── talk.html          # Event + workPerformed: ScholarlyArticle
│       ├── publication.html   # ScholarlyArticle + DOI/PubMed/PMC identifiers
│       ├── preprint.html      # ScholarlyArticle (preprint status) + arXiv identifier
│       └── breadcrumb.html    # BreadcrumbList for sub-pages
├── _layouts/
│   ├── default.html           # Base layout (sets html lang)
│   ├── talk.html, publication.html, preprint.html
├── _talks/                    # Output:true → individual detail pages
├── _publications/, _preprints/  # Output:true → individual detail pages
├── _awards/                   # Output:false → fed into Person.award array
├── _writings/                 # Output:false → WRITINGS section (industry magazines, commentary)
├── _templates/                # Scaffolds (excluded from build); copy to add new content
├── assets/                    # css, js, img
├── index.html                 # Homepage
├── 404.html                   # Custom 404
├── manifest.webmanifest       # PWA manifest
├── feed.xml                   # Combined Atom feed (all content collections)
├── publications.bib           # Auto-generated BibTeX export
└── robots.txt                 # SEO directives
```

`/sitemap.xml` is auto-generated by `jekyll-sitemap`.

## Adding content

Copy the matching template from `_templates/` into the right collection folder. **Layouts are auto-applied via `_config.yml` defaults — you do not need a `layout:` line.**

| Want to add | Folder | Template | Auto-generated |
|---|---|---|---|
| Talk / presentation | `_talks/YYYY-MM-DD-slug.md` | `_templates/talk.md` | `Event` + nested `ScholarlyArticle` JSON-LD, `citation_*`, breadcrumb, sitemap, feed |
| Journal article | `_publications/YYYY-MM-DD-slug.md` | `_templates/article.md` | `ScholarlyArticle` with DOI/PubMed/PMC, `citation_*`, breadcrumb, sitemap, feed |
| Conference paper | `_publications/YYYY-MM-DD-slug.md` | `_templates/conference.md` | Same as article, listed under CONFERENCE PAPERS |
| Preprint | `_preprints/YYYY-MM-DD-slug.md` | `_templates/preprint.md` | `ScholarlyArticle` (preprint status) with arXiv id, `citation_*`, breadcrumb, sitemap, feed |
| Award | `_awards/YYYY-MM-DD-slug.md` | `_templates/award.md` | Appended to `Person.award[]` and homepage AWARDS section |
| Writing (commentary, magazine, essay) | `_writings/YYYY-MM-DD-slug.md` | `_templates/writing.md` | Listed under WRITINGS section + Recent Works feed; supports `subtitle:` (English subtitle for non-English titles) and `link:` (external URL once published) |

The detail page, sitemap entry, feed entry, JSON-LD, and (for academic content) Google Scholar `citation_*` meta tags are emitted automatically — no template changes required.

### Updating identity / social

Edit `_data/profile.yml`. The `social:` array drives both:
- The CONTACT section icons (`_includes/social-icon.html` — supported platforms: `linkedin`, `x`, `github`, `orcid`, `scholar`, `researchgate`)
- The `Person.sameAs` array in the homepage JSON-LD

Adding a new social URL requires only adding an entry to `social:`. To support a new platform icon, add a `{% when %}` case in `_includes/social-icon.html`.

## SEO / search engine setup

After deploying:
1. **Google Search Console** — verify with `google33d86ef7752ab266.html` (already present), submit `sitemap.xml`, request indexing for the homepage
2. **Google Scholar** — automatic discovery via `citation_*` meta on talk/publication/preprint pages
3. **Rich Results Test** — validate at <https://search.google.com/test/rich-results>

## Plugins (whitelisted on GitHub Pages)

- `jekyll-seo-tag` — emits title/description/og/twitter meta + base JSON-LD
- `jekyll-sitemap` — auto-generates `/sitemap.xml`
- `jekyll-redirect-from` — supports URL aliases via front matter `redirect_from:`

## Local development

```bash
gem install bundler jekyll
bundle exec jekyll serve
# → http://localhost:4000
```

## Deployment

Push to `main` → GitHub Pages builds and deploys (1–2 min).
